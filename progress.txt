### KEY DESIGN DECISIONS
- checkWithContext method added alongside existing check (backward compatible)
- Rules with checkWithContext get called preferentially; rules without it fall back to check
- Taint maps built lazily per-file (only when at least one rule needs data-flow)
- SecurityConfig supports both { security: { ... } } wrapper and flat format
- CLI overrides always win over file config
- Path validation detection (path.resolve/normalize) suppresses path traversal findings
- eval() with string literal downgraded to info severity (not a real risk)
